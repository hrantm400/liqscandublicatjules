# Обновления системы — 12 Марта 2026

Сегодня мы реализовали ряд важных исправлений интерфейса (UI) и новую функцию ручного выбора часового пояса. Ниже представлен подробный список изменений:

## 1. Функция ручного выбора часового пояса (Timezone Selection)
- **База данных и Backend**: Добавлено поле `timezone` в модель `User` (Prisma) и обновлен API для его поддержки.
- **Окно приветствия (Onboarding Gate)**: Новые пользователи теперь обязаны выбрать свой часовой пояс при первом входе. Для старых пользователей добавлен специальный баннер с просьбой обновить настройки.
- **Настройки профиля**: В раздел `Settings.tsx` и `Profile.tsx` добавлен удобный выпадающий список `TimezoneSelector` для изменения часового пояса в любой момент.
- **Конвертация времени (App-wide Timestamp Conversion)**: 
  - Создан единый компонент `TimeDisplay.tsx` для автоматической конвертации времени сигналов из UTC в локальное время пользователя.
  - Применено ко всем таблицам сигналов (SuperEngulfing, ICT Bias, RSI, CRT, Strategy1) и карточкам для мобильных устройств.
  - В интерактивном графике обновлены тултипы (и сами маркеры) для корректного отображения локального времени сигналов при наведении.

## 2. Исправления графиков (Chart Alignment & Performance)
- **Пустое пространство справа (Chart Alignment)**: Настроена зона видимости (zoom) при открытии основного графика. Теперь свеча сигнала находится левее, комфортно оставляя 30-40% открытого пустого пространства с правой стороны для анализа.
- **Быстродействие (Performance)**: Уменьшено максимальное количество запрашиваемых свечей при старте с 500 до 300, что моментально ускорило время загрузки графика без потери информативности.
- **Мини-графики (Mini Charts)**: Исправлено центрирование и пропорции отображения (`preserveAspectRatio="xMidYMid meet"`) на маленьких карточках.

## 3. Фильтры и статистика (Filters & Stats)
- **Статистические карточки (Timeframes & Session Stats)**: Теперь большие карточки со статистикой на страницах стратегий жестко привязаны к активной вкладке. Переключение между "Live" и "Closed" динамически корректирует цифры.
- **Фильтр по объему (Volume Filter)**: Страница Dashboard («Monitor Overview») теперь честно применяет фильтр в $20M до того, как подсчитывает количество активных сигналов.
- **Фильтр по направлению (SIDE Filter: All/Longs/Shorts)**: Исправлена логика передачи состояний `directionFilter`, теперь сортировка шортов/лонгов снова работает корректно.
- **Фильтр по рангу (Ranking Filter)**: Изменена логика: теперь монеты фильтруются по их реальному рыночному рангу (CMC Rank), а не просто обрезанием массива.
- **Статус-вкладка (Default Tab)**: Вкладкой по умолчанию на всех страницах монитора окончательно закреплена "Live Signals" (раньше по ошибке открывались все).
- **Чистка меню (FilterMenu)**: Убрана нерабочая и дублирующая панель статуса (Won / Lost / Expired) из бокового Filter Menu, так как она дублировала `StatusTabs`.

## 4. Навигация и код
- Использование кнопок **Next/Prev** в деталях сигнала теперь работает даже после обновления (F5) страницы! Добавлен механизм сохранения контекста последней таблицы сигналов напрямую в память (`sessionStorage`).
- Произведена чистка неиспользуемых переменных, исправлены все ошибки линтера. Сборка фронтенда проходит за 8 секунд со 100% успехом.
